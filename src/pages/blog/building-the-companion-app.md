The companion app is the book's final payoff in one small screen: ask an engineering
question against the sample Daily Drilling Report archive and watch the whole idea run
end to end — **retrieval → local-model answer → citations → the original evidence**.

## Design principle: evidence first, answer second

The app deliberately shows the retrieved reports — with scores, dates, and excerpts —
alongside every generated answer. If the evidence doesn't contain the answer, the prompt
tells the model to say so rather than guess. A "Why this answer?" transparency panel
exposes everything behind the response: the ranked reports, the evidence snippets, and
the exact prompt sent to the local model.

## Nothing new, just assembled

The app doesn't reimplement anything. Retrieval comes from Chapter 4's semantic search,
generation from Chapter 5's RAG loop, and the archive handling from Chapter 9 — imported,
not rewritten. The Streamlit file is only the screen around the book's own code. That's
the point: by the time you reach it, you've already built every piece yourself.

## Local by default

The embedding model (`all-MiniLM-L6-v2`) runs on CPU, and answers are generated by a
local Ollama model (`qwen2.5:7b-instruct` by default). If Ollama isn't running, the app
says so plainly and shows the retrieved evidence anyway — retrieval never depends on the
LLM. Nothing leaves your machine.

## Try it

```bash
pip install -r book/requirements.txt
ollama pull qwen2.5:7b-instruct   # optional, for generated answers
streamlit run book/app/streamlit_app.py
```

It runs on the ten-report public Utah FORGE sample archive that ships with the
repository. Screenshots are on the [demo page](/demo/), and the source is in
[book/app on GitHub](https://github.com/djimrastephane/ddr-rag-book/tree/main/book/app).
