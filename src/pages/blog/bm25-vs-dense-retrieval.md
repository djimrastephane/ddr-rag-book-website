BM25 is unbeatable when the query contains an exact code, depth, or part number. Dense
embeddings win when the query and the report describe the same thing in different words.
Engineering archives need both.

## The short version

- **BM25** — precise, fast, transparent; fails on vocabulary mismatch.
- **Dense retrieval** — finds meaning, not words; can miss exact identifiers.
- **Hybrid** — fuse both scores and get the best of each (Chapter 9 of the book).
